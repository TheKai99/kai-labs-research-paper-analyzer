**AI-powered research paper analysis built with Django & Google Gemini.**

KAI Labs is an AI-powered research paper analyzer designed to help students and researchers understand academic papers faster.

The platform allows users to upload research papers in PDF format and automatically generate structured summaries using artificial intelligence. Users can also ask questions about the paper, compare two research papers, and search for related research online.

The goal of this project is to simplify academic research analysis and make complex research papers easier to understand.

## 🚀 Getting Started

### 1. Clone the Repository

```bash
git clone https://github.com/TheKai99/kai-labs-research-paper-analyzer.git
cd kai-labs-research-paper-analyzer
```

### 2. Install Dependencies

```bash
pip install -r requirement.txt
```

### 3. Set Up Environment Variables

Rename `_env` to `.env` inside the `project/` folder and add your Gemini API key:

```
GEMINI_API_KEY=your_gemini_api_key_here
```

> 🔑 Get your free API key at [aistudio.google.com](https://aistudio.google.com)

### 4. Apply Migrations & Run

```bash
cd project
python manage.py migrate
python manage.py runserver
```

Then open **http://127.0.0.1:8000** in your browser.

## 🔄 Execution Flow

```
.env  ──────────────────►  Loads GEMINI_API_KEY securely
                                      │
requirement.txt  ──────►  All dependencies installed
                                      │
manage.py  ─────────────►  Django app starts
                                      │
db.sqlite3  ────────────►  Database ready (stores results)
                                      │
User uploads PDF  ──────►  Django view processes it
                                      │
Gemini API  ────────────►  AI analyzes the research paper
                                      │
Response  ──────────────►  Results displayed to user
```

## ⚠️ Important Notes

- **Never upload your `.env` file** to GitHub — it contains your secret API key
- Add `.env` and `db.sqlite3` to your `.gitignore`
- The `_env` file in this repo is a template — rename it to `.env` before running
